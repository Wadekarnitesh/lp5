================================================================================
                    SECTION 1: WHAT IS LINEAR REGRESSION?
================================================================================

DEFINITION:
Linear Regression is a fundamental supervised machine learning algorithm used 
to predict a continuous target variable based on one or more input features. 
It assumes a linear relationship between the input variables (features) and 
the single output variable (target).

MATHEMATICAL REPRESENTATION:

Simple Linear Regression (Single Feature):
    y = mx + b
    or
    y = ОІвӮҖ + ОІвӮҒx

Where:
    y = predicted value (dependent variable)
    x = input feature (independent variable)
    ОІвӮҖ = intercept (y-intercept)
    ОІвӮҒ = slope (coefficient)
    m = slope of the line

Multiple Linear Regression (Multiple Features):
    y = ОІвӮҖ + ОІвӮҒxвӮҒ + ОІвӮӮxвӮӮ + ОІвӮғxвӮғ + ... + ОІвӮҷxвӮҷ

Where:
    y = predicted value
    xвӮҒ, xвӮӮ, xвӮғ...xвӮҷ = input features
    ОІвӮҖ = intercept term
    ОІвӮҒ, ОІвӮӮ, ОІвӮғ...ОІвӮҷ = coefficients for each feature

ASSUMPTIONS OF LINEAR REGRESSION:

1. Linearity: The relationship between features and target is linear
2. Independence: Observations are independent of each other
3. Homoscedasticity: Constant variance of residuals
4. Normality: Residuals are normally distributed
5. No Multicollinearity: Independent features are not highly correlated

COST FUNCTION (MEAN SQUARED ERROR - MSE):
    J(ОІ) = (1/2m) ОЈ(hбөҰ(xвҒұ) - yвҒұ)ВІ

Where:
    m = number of training examples
    hбөҰ(xвҒұ) = predicted value for i-th example
    yвҒұ = actual value for i-th example

GOAL OF LINEAR REGRESSION:
Minimize the cost function (MSE) by finding optimal values of ОІвӮҖ, ОІвӮҒ...ОІвӮҷ

                    SECTION 3: CONCEPT OF DEEP NEURAL NETWORK
================================================================================

DEFINITION:
A Deep Neural Network (DNN) is an artificial neural network with multiple 
hidden layers between the input and output layers. It is called "deep" because 
it has more than one hidden layer, allowing it to learn hierarchical 
representations of data.

BASIC COMPONENTS OF DNN:

1. NEURONS (NODES):
   Basic computational units that receive inputs, process them, and produce outputs
   Each neuron has:
   - Input connections with weights (wвӮҒ, wвӮӮ, ..., wвӮҷ)
   - A bias term (b)
   - An activation function (f)
   - Output connection

2. LAYERS:
   a. Input Layer: Receives raw data features
      - Number of neurons equals number of features
      - No computation performed, just passes data forward
   
   b. Hidden Layers: Perform computations and learn patterns
      - Can be one or more layers
      - Each layer learns increasingly complex features
      - First hidden layer: Simple patterns
      - Deeper layers: Complex, abstract patterns
   
   c. Output Layer: Produces final prediction
      - For regression: Single neuron with linear activation
      - For classification: Multiple neurons with softmax activation

3. WEIGHTS:
   - Parameters that determine the strength of connections between neurons
   - Randomly initialized, updated during training
   - Represent the "knowledge" learned by the network

4. BIASES:
   - Additional parameters that allow shifting activation functions
   - Help models fit data better by providing flexibility

5. ACTIVATION FUNCTIONS:
   Common activation functions and their uses:
   
   a) ReLU (Rectified Linear Unit):
      f(x) = max(0, x)
      - Most commonly used for hidden layers
      - Overcomes vanishing gradient problem
      - Computationally efficient
   
   b) Sigmoid:
      f(x) = 1 / (1 + eвҒ»ЛЈ)
      - Output range (0, 1)
      - Used for binary classification output
   
   c) Tanh (Hyperbolic Tangent):
      f(x) = (eЛЈ - eвҒ»ЛЈ) / (eЛЈ + eвҒ»ЛЈ)
      - Output range (-1, 1)
      - Zero-centered, better than sigmoid
   
   d) Linear (Identity):
      f(x) = x
      - Used for regression output layer
      - No transformation applied

DETAILED WORKING PROCESS OF DNN:

PHASE 1: FORWARD PROPAGATION

Forward propagation is the process of passing input data through the network 
to generate predictions. Here's the step-by-step process:

Step 1 - Input Reception:
   The input layer receives feature values (xвӮҒ, xвӮӮ, ..., xвӮҒвӮғ)
   For Boston Housing: RM=6.5, LSTAT=15%, PTRATIO=18, etc.
   No computation happens at input layer - data is just passed forward

Step 2 - Hidden Layer 1 Computation:
   For each neuron j in first hidden layer:
   
   a) Calculate weighted sum (Z):
      Zвұј = wвӮҒвұјxвӮҒ + wвӮӮвұјxвӮӮ + ... + wвӮҒвӮғвұјxвӮҒвӮғ + bвұј
   
   b) Apply activation function (ReLU):
      Aвұј = ReLU(Zвұј) = max(0, Zвұј)
   
   This creates 64 outputs (AвӮҒ through AвӮҶвӮ„)

Step 3 - Hidden Layer 2 Computation:
   For each neuron k in second hidden layer:
   
   a) Weighted sum using outputs from previous layer:
      ZвӮ– = wвӮҒвӮ–AвӮҒ + wвӮӮвӮ–AвӮӮ + ... + wвӮҶвӮ„вӮ–AвӮҶвӮ„ + bвӮ–
   
   b) Apply ReLU activation:
      AвӮ– = ReLU(ZвӮ–)
   
   This creates 32 outputs (AвӮҒ through AвӮғвӮӮ)

Step 4 - Output Layer Computation:
   For the single output neuron:
   
   a) Weighted sum using outputs from second hidden layer:
      Z_out = wвӮҒAвӮҒ + wвӮӮAвӮӮ + ... + wвӮғвӮӮAвӮғвӮӮ + b_out
   
   b) Apply linear activation (no transformation):
      yМӮ = Z_out (predicted house price)

PHASE 2: LOSS CALCULATION

After forward propagation, we calculate how wrong our prediction is:

Loss Function (Mean Squared Error - MSE):
    Loss = (1/2) Г— (yМӮ - y)ВІ

Where:
    yМӮ = predicted price
    y = actual price

For batch processing:
    Batch Loss = (1/2m) Г— ОЈ(yМӮвҒұ - yвҒұ)ВІ

Example calculation:
    If actual price = $350,000 (scaled to 350)
    Predicted price = $325,000 (scaled to 325)
    MSE = (1/2) Г— (325 - 350)ВІ = (1/2) Г— (-25)ВІ = 312.5

PHASE 3: BACKPROPAGATION

Backpropagation calculates how each weight and bias contributed to the error 
and computes gradients for updating parameters.

Step 1 - Output Layer Gradient:
    вҲӮL/вҲӮyМӮ = (yМӮ - y)
    This represents how much the output affects the loss

Step 2 - Output Layer Weights Gradient:
    вҲӮL/вҲӮw_out = вҲӮL/вҲӮyМӮ Г— вҲӮyМӮ/вҲӮw_out = (yМӮ - y) Г— A_hidden

Step 3 - Hidden Layer 2 Gradients:
    Error propagates backward through the network
    For each neuron: ОҙвӮ– = (w_out Г— Оҙ_out) Г— ReLU_derivative(ZвӮ–)

Step 4 - Hidden Layer 1 Gradients:
    Оҙвұј = ОЈ(wвӮ–вұј Г— ОҙвӮ–) Г— ReLU_derivative(Zвұј)

PHASE 4: WEIGHT UPDATES (OPTIMIZATION)

Using the Adam Optimizer (used in our implementation):

Adam combines two concepts:
1. Momentum: Accumulates past gradients to smooth updates
2. RMSprop: Adapts learning rates for each parameter

Update Rule for each parameter Оё (weight or bias):

    mвӮң = ОІвӮҒ Г— mвӮңвӮӢвӮҒ + (1 - ОІвӮҒ) Г— gвӮң          (Momentum term)
    vвӮң = ОІвӮӮ Г— vвӮңвӮӢвӮҒ + (1 - ОІвӮӮ) Г— gвӮңВІ         (RMSprop term)
    
    mМӮвӮң = mвӮң / (1 - ОІвӮҒбө—)                    (Bias correction)
    vМӮвӮң = vвӮң / (1 - ОІвӮӮбө—)                    (Bias correction)
    
    ОёвӮң = ОёвӮңвӮӢвӮҒ - Оұ Г— mМӮвӮң / (вҲҡvМӮвӮң + Оө)

Where:
    gвӮң = current gradient
    Оұ = learning rate (0.001 default)
    ОІвӮҒ = 0.9 (decay rate for momentum)
    ОІвӮӮ = 0.999 (decay rate for RMSprop)
    Оө = 1e-8 (small constant for numerical stability)

COMPLETE TRAINING ITERATION EXAMPLE:

Initial State (Epoch 1, Batch 1):
    Input: RM=6.5, LSTAT=15%, PTRATIO=18
    Actual Price: 350
    Random initial weights and biases

Forward Pass:
    ZвӮҒ = wвӮҒГ—6.5 + wвӮӮГ—15 + wвӮғГ—18 + bвӮҒ
    AвӮҒ = max(0, ZвӮҒ)
    ... (computations continue)
    yМӮ = 325

Loss Calculation:
    Loss = 312.5

Backward Pass:
    Calculate gradients for all weights
    вҲӮL/вҲӮw_out = -12.5
    вҲӮL/вҲӮwвӮҒ = -0.85, etc.

Update Weights:
    new_w_out = old_w_out - 0.001 Г— (momentum-adjusted gradient)
    All weights get updated slightly

After 50 Epochs (Full training process):
    Loss decreases from ~312 to ~18
    Predictions become much more accurate
    Final prediction for same input: yМӮ = 348 (close to actual 350)

EPOCHS AND BATCHES EXPLANATION:

1. Epoch:
   One complete pass through the entire training dataset
   Example: 404 training samples = 1 epoch

2. Batch:
   Subset of training data processed together
   Batch size = 32 means we process 32 samples before updating weights

3. Iteration:
   One batch processed = one weight update
   404 samples / 32 batch size вүҲ 13 iterations per epoch
