SECTION 15: MODEL EVALUATION METRICS
================================================================================

Lines 152-171: Calculate and display evaluation metrics

Line 154-155: test_mse = mean_squared_error(y_test, y_pred)
- Formula: MSE = (1/n) × Σ(y_actual - y_pred)²
- Penalizes large errors more than small ones
- Lower value = Better model

Line 157-158: test_mae = mean_absolute_error(y_test, y_pred)
- Formula: MAE = (1/n) × Σ|y_actual - y_pred|
- More robust to outliers than MSE
- Easy to interpret (same units as target)

Line 160: test_rmse = np.sqrt(test_mse)
- Root Mean Squared Error
- Same units as target variable
- Most interpretable metric for pricing errors

Line 162-163: test_r2 = r2_score(y_test, y_pred)
- Formula: R² = 1 - (Sum of squared residuals / Total sum of squares)
- Range: 0 to 1 (higher is better)
- Interpreted as variance explained by model

================================================================================
NEURAL NETWORK FUNCTIONS DETAILED EXPLANATION
================================================================================

FUNCTION 1: Dense Layer
- Syntax: Dense(units, activation, input_shape)
- Purpose: Fully connected layer where each neuron connects to all inputs
- Working: y = activation(W·x + b)
  * W: Weight matrix
  * x: Input vector
  * b: Bias vector
- Importance: Learns complex non-linear relationships

FUNCTION 2: ReLU Activation
- Formula: ReLU(x) = max(0, x)
- Purpose: Introduces non-linearity
- Advantages:
  * Computationally efficient
  * Mitigates vanishing gradient problem
  * Sparse activation (many zeros)
- Importance: Enables learning of complex patterns

FUNCTION 3: Adam Optimizer
- Full name: Adaptive Moment Estimation
- Working: Combines Momentum and RMSprop
- Features:
  * Adaptive learning rate per parameter
  * Momentum for faster convergence
  * Bias correction for initial steps
- Parameters: β1=0.9, β2=0.999, learning_rate=0.001
- Importance: Best general-purpose optimizer

FUNCTION 4: Mean Squared Error Loss
- Formula: MSE = (1/n) Σ(y_pred - y_actual)²
- Properties:
  * Always positive
  * Squared units (harder to interpret)
  * Penalizes outliers heavily
- Importance: Smooth loss function for optimization

================================================================================
DATASET FEATURES EXPLANATION
================================================================================

Feature meanings (Boston Housing Dataset):

1. CRIM: Crime rate per capita by town
2. ZN: Proportion of residential land zoned for large lots
3. INDUS: Proportion of non-retail business acres
4. CHAS: Charles River dummy (1 if tract bounds river)
5. NOX: Nitric oxides concentration (pollution)
6. RM: Average number of rooms per dwelling
7. AGE: Proportion of owner-occupied units built before 1940
8. DIS: Weighted distances to Boston employment centers
9. RAD: Index of accessibility to radial highways
10. TAX: Full-value property tax rate
11. PTRATIO: Pupil-teacher ratio by town
12. B: 1000(Bk - 0.63)² where Bk is proportion of Black population
13. LSTAT: Percentage of lower status population
14. PRICE (Target): Median value of owner-occupied homes in $1000s

================================================================================
TROUBLESHOOTING NOTES
================================================================================

Common Issues and Solutions:

1. Data Loading Error:
   - Internet connection required for URL
   - Alternative: Download CSV to local file

2. Shape Mismatch:
   - Verify X has 13 features
   - Check X_train shape after scaling

3. Poor Model Performance:
   - Try more epochs (100-200)
   - Add more layers or neurons
   - Adjust learning rate
   - Add dropout for regularization

4. Overfitting:
   - Increase validation split
   - Add Dropout layers
   - Reduce model complexity
   - Use early stopping

5. Scalar Issues:
   - Must fit on training data only
   - Transform test data separately
   - Scale new predictions before using
