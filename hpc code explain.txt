SECTION 7: PARALLEL REDUCTION (THE CORE)
The Pragma Directive
c
#pragma omp parallel for reduction(+:sum) reduction(min:min) reduction(max:max)
This is the heart of parallel computing in this program. Let me break it down:

Component	Explanation
#pragma omp	OpenMP compiler directive prefix
parallel	Creates a team of threads
for	Distributes loop iterations among threads
reduction(+:sum)	Sum operation: private copy initialized to 0, combined with +
reduction(min:min)	Min operation: private copy initialized to MAX, combined with min
reduction(max:max)	Max operation: private copy initialized to MIN, combined with max
Serial vs Parallel Execution Comparison
Aspect	SERIAL (without pragma)	PARALLEL (with pragma)
Number of threads	1	Multiple (default = CPU cores)
min variable	Single shared variable	Each thread has private copy
max variable	Single shared variable	Each thread has private copy
sum variable	Single shared variable	Each thread has private copy
Race condition	Not applicable (single thread)	PREVENTED by reduction clause
Execution speed	Slow for large SIZE	Fast (near N/threads time)
Correctness	Always correct	Always correct (reduction ensures)
What Happens During Parallel Execution:
text
Initial State:
Thread 0 gets private copy: sum0=0, min0=INT_MAX, max0=INT_MIN
Thread 1 gets private copy: sum1=0, min1=INT_MAX, max1=INT_MIN
Thread 2 gets private copy: sum2=0, min2=INT_MAX, max2=INT_MIN
Thread 3 gets private copy: sum3=0, min3=INT_MAX, max3=INT_MIN

Work Distribution (Example with 4 threads):
Thread 0 processes indices 0-24999
Thread 1 processes indices 25000-49999
Thread 2 processes indices 50000-74999
Thread 3 processes indices 75000-99999

After Loop:
Thread 0 result: sum0, min0, max0
Thread 1 result: sum1, min1, max1
Thread 2 result: sum2, min2, max2
Thread 3 result: sum3, min3, max3

Reduction (Combining):
Final sum = sum0 + sum1 + sum2 + sum3
Final min = min(min0, min1, min2, min3)
Final max = max(max0, max1, max2, max3)
